# 어드바이저 스킬

devex는 사상·템플릿(엔진)만 담고 실제 방향·데이터는 소비 프로젝트가 채운다. 이 스킬은 소비 프로젝트가 공급한 **방향 프로파일**로 산출물을 비판하거나 성장 방향을 조언한다. 방법론을 제정하지 않고 PARA 골격을 쓴다.

## 호출 방식

- `/advisor review <대상>`: 산출물을 방향·렌즈로 비판
- `/advisor advise`: 방향 대비 다음 액션 조언
- 특정 렌즈만 적용하려면 렌즈 이름을 지정해 호출한다(미지정 시 프로파일의 전체 렌즈 적용).
- 트리거 키워드: "어드바이저", "방향 검토", "성장 조언"

## 방향 프로파일 연동 (계약)

프로파일은 유저 스코프(`~/.claude/advisor/profiles/*.json`)에 두어 플러그인 업데이트와 독립 유지한다. 소비 프로젝트가 자기 방향·렌즈·참조·PARA 경로를 채운다.

### 탐색 절차

```bash
# 1. 유저 스코프
ls ~/.claude/advisor/profiles/*.json 2>/dev/null
# 2. 없으면 프로젝트 스코프
ls config/advisor/profiles/*.json 2>/dev/null
```

스키마는 `config/advisor/profiles/example.json` 참조. 필수 `direction`·`lenses`, 선택 `references`·`para`. 프로파일이 여러 개면 `project` 필드로 현재 프로젝트와 매칭하고, 하나면 그것을 쓰며, 없으면 example 을 복사해 채우도록 안내하고 멈춘다(추측 금지). 현재 방법론 골격은 PARA만 지원한다.

## 모드

- **review**: 대상을 각 `lens.focus`로 통과시켜 신호 대 노이즈·방어 가능성·방향 정렬·순진한 프레이밍·근거 없는 단정을 잡는다. 리스크·비검증 상세는 카드로 분리 제안(삭제 아님).
- **advise**: 방향(direction) 대비 현재 PARA의 Projects·Areas 정렬을 보고 다음 액션을 제안한다.

## PARA 연동

`templates/para/README.md`의 골격을 따른다. 데이터는 소비 프로젝트가 `para.root` 아래 버킷에 둔다. 엔진은 데이터를 소유하지 않는다.

## 실행

깊은 비판·조언이 필요하면 `advisor` 에이전트(`agents/advisor.md`)로 위임한다. 에이전트 결과는 사용자에게 직접 보이지 않으므로 호출자가 완료 즉시 전달한다.

## 경계

- 자동 수정 금지. 검토·제안만.
- 데이터·개인 정보는 담지 않는다. 방향·데이터는 소비 프로젝트 스코프.

### 다른 검증과의 역할 경계

| 도구 | 무엇을 보나 | advisor와 다른 점 |
|------|-------------|-------------------|
| `content-verify` | 문서 품질(사실·구조·가독성·톤·AI 티) | 방향·타깃 무관, 문서 자체 품질만 |
| `cross-verify` | 의사결정·설계·문서·구현 4축 정합 | 산출물 내부 정합, 성장 방향 개념 없음 |
| `advisor` | 선언된 방향·렌즈 관점의 비판·조언 | 소비 프로젝트의 방향 프로파일 기준 판단 |

세 도구를 함께 쓸 때 같은 문장에 중복 지적이 나오면, 호출 정책(순서·적용 대상)은 소비 프로젝트가 자기 프로파일·규칙에서 정한다.
