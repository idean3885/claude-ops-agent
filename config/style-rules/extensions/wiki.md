# 위키 익스텐션

> **Base**: `base/readability.md` + `base/tone.md` + `base/punctuation.md` + `base/ai-tells.md`
> **Adds**: 사내 위키 포스팅 특화 규칙 (시리즈 인덱스, 상세도, 사내 맥락 보존)

사내 위키는 블로그 발행의 **원본 저장소**다.
"내가 더 잘 이해하고 업무를 잘하기 위한 저장소" 라는 1차 목적을 가진다.
대외비 제거·일반화는 위키 작성 시점이 아니라 블로그 발행 시점에 `devex:content-publish`가 처리한다.

---

## 적용 대상

| 매체 | 적용 |
|------|------|
| Dooray 사내 위키 | 본 익스텐션 적용 (`toolkit:content-write` 호출) |
| 블로그 발행 전 원본 | 본 익스텐션 → 발행 시 대외비 마스킹 거쳐 `blog.md` 익스텐션으로 전환 |

---

## W1. 상세도 우선 — 원본은 압축하지 않는다

위키 원본은 포트폴리오의 원천 데이터다. 상세할수록 좋다.

| 글 유형 | 위키 원본 | 블로그 | PAAR 적용 |
|---------|----------|--------|----------|
| 시리즈·프로젝트 | 스펙·대안·트레이드오프·수치·출처까지 상세 | 읽히는 수준으로 압축 | **필수** |
| 개발 사전·노하우 | 실무 판단이 포함된 개념 정리 | 경량 가공 후 발행 | **경량** (`knowledge.md` 참조) |

블로그 발행 시 변환 비용은 발생하지만, 발행 오케스트레이터가 자동 처리한다.

---

## W2. 사내 맥락 보존

위키 작성 시점에는 대외비 제거를 수행하지 않는다.
사내 시스템명·프로젝트 코드명·내부 URL·구성원명·구체 수치를 그대로 기록한다.

이유:
- 작성 시점 대외비 제거는 인지 부하 증가
- 사내 맥락 손실 시 실무 참조 가치 하락
- 발행 단계에서 `~/.claude/devex/confidential-keywords.local.json` 기반 자동 마스킹

---

## W3. 시리즈 인덱스 = 탐색 허브

위키의 시리즈 상위 페이지는 블로그의 인덱스 포스팅과 **역할이 다르다**.

| 역할 | 위키 인덱스 | 블로그 인덱스 (`blog.md B3`) |
|------|-----------|--------------------------|
| 목적 | 시리즈 하위 페이지 탐색 | 이력서·포트폴리오 랜딩 |
| 내용 | 시스템 개요 + 하위 목록 | 프로젝트 요약 + 각 편 핵심 + 링크 |
| 독자 | 본인 (위키 탐색용) | 면접관 (1분 내 전체 파악용) |
| 갱신 | 하위 포스팅 추가 시 | 시리즈 현행화 시 합본 역할 |

---

## W4. 본문 H1 제거 (Dooray 위키 API 제약)

Dooray 위키는 `subject` 필드가 페이지 제목으로 표시된다.
본문(`content`)이 `# 제목` 으로 시작하면 제목이 중복 노출된다.

본문 작성 시 H1(`# ...`)을 제거하고 H2(`## ...`)부터 시작한다.
`readability.md H1` (제목은 하나만) 과 자연스럽게 맞는다.

---

## W5. 블로그 메타 항상 작성

위키 포스팅은 블로그 발행의 원본이므로, 발행 여부와 무관하게 **항상 블로그 메타를 작성**한다.

```markdown
---

> **블로그 메타**
> 카테고리: {메인 카테고리}, {서브 카테고리}
> 태그: {태그1}, {태그2}, ...
> 블로그 발행: 미발행 | 발행 ({날짜}, {URL})
> 위키 발행: 미발행 | 발행 ({대상 위키명})
```

블로그 카테고리·태그 목록은 `toolkit:content-write` 가 블로그 레포에서 조회해 제시한다.

---

## W6. 독립 가치 점검 (시리즈 편 추가 시)

새 편 추가 시 다음 질문을 통과해야 한다.

- 이 글에서 AI가 쓸 수 없는 문장이 3개 이상인가?
- 시리즈 맥락 없이 이 글 단독으로 가치가 있는가?
- 같은 주장이 3회 이상 다른 섹션에서 반복되는가? (반복 = 제거)

3개 미만이면 분석(Analyze) 보강이 필요하다.

---

## 필수 구조 (시리즈·프로젝트)

위키 원본은 발행 시 압축되지만, 원본에는 상세 섹션을 모두 유지한다.

```markdown
## TL;DR
## 배경 (사내 맥락 포함, 발행 시 마스킹)
## 분석 (대안·트레이드오프·근거)
## 해결 (구현 상세, 사내 시스템명 그대로)
## 결과 (수치)
## 회고
## 참고 (사내 위키 링크·구성원·티켓 ID)

---

> **블로그 메타** (W5)
```

## base 대비 적용 강도

| 규칙 | base | 위키 | 비고 |
|------|------|------|------|
| K3 (외래어 표기) | 부분 판별 | **필수** | 사내 위키도 외부 블로그 발행 원본이라 동일 |
| V3 (TL;DR) | 권장 | **필수** | 위키도 본인 재참조용으로 TL;DR 필수 |
| H1 (단일 H1) | 가능 판별 | **본문 H1 금지** | W4 (Dooray 위키 API 제약) |
| ai-tells 전체 | 차단 | 차단 (단 사내 용어는 Do-NOT) | 사내 시스템명·약어는 마스킹 대상이지 AI 티 아님 |

## 합격선

- 위키 원본은 블로그 합격선보다 느슨하다 (작성 빈도 우선)
- B 등급 이상이면 발행 후보. 발행 시점에 `devex:content-publish`가 A 등급으로 가공
