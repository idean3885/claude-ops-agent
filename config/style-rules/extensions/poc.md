# PoC 익스텐션

> **Base**: `base/readability.md` + `base/tone.md` + `base/punctuation.md` + `base/ai-tells.md`
> **Adds**: PoC 문서 특화 규칙 (결론 선행 + 재현 가능성)

PoC 문서는 기술 검증의 결과를 "된다·안 된다"로 명확히 결론짓고, 동일 환경에서 재현 가능한 절차를 남기는 것이 목적이다.

---

## 적용 대상

| 매체 | 적용 |
|------|------|
| 사내 PoC 문서 (위키·프로젝트 레포 `.docs/`) | 본 익스텐션 |
| 블로그 PoC 회고 | `blog.md` 익스텐션 + 본 익스텐션의 결론 선행·재현 가능성 항목 |

---

## PC1. 결론 선행 (TL;DR 필수)

TL;DR 에 다음 3가지를 명시한다.

- **판정**: 된다·안 된다·조건부 (한 단어)
- **핵심 수치**: 검증 결과의 핵심 지표 (응답 시간·메모리 등)
- **적용 범위·한계**: 어떤 환경에서 성립하는지

```markdown
## TL;DR

- **판정**: 조건부 (JDK 21+, Spring Boot 3.2+)
- **수치**: 동시성 1000 요청 처리, p99 240ms (기존 380ms 대비 37% 감소)
- **한계**: ThreadLocal 기반 라이브러리(MDC 등)와 충돌. 우회책 별도 PoC 예정
```

## PC2. 재현 가능성 — 환경 명시

환경 섹션에는 동일 검증을 재현하는 데 필요한 모든 정보를 기재한다.

```markdown
## 환경

- JDK: 21.0.2 (Temurin)
- Spring Boot: 3.2.3
- 데이터베이스: PostgreSQL 16 (로컬 docker)
- 부하 도구: k6 0.49.0
- 설정: `application.yml` 전문 (코드 블록)
```

## PC3. 코드 → 결과 → 해석 순서

PoC 본문은 세 구간의 반복이다.

```markdown
### 시도 1: virtual thread 활성화만

(코드)
```yaml
spring.threads.virtual.enabled: true
```

(결과)
```
p99: 320ms
처리량: 850 req/s
```

(해석)
처리량은 개선됐지만 p99 가 충분히 떨어지지 않았습니다. 다음 변수로 ThreadLocal 사용처를 의심합니다.
```

## PC4. 결론 섹션 = 판단 근거 + 다음 단계

```markdown
## 결론

- 도입 가능 여부: 조건부 도입
- 판단 근거: TL;DR 수치 + ThreadLocal 충돌 영향 범위 추정
- 제약사항: MDC·SecurityContext 영향
- 다음 단계: ThreadLocal 우회 PoC, 메모리 프로파일링
```
