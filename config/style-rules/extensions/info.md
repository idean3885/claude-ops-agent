# 정보성 익스텐션

> **Base**: `base/readability.md` + `base/tone.md` + `base/punctuation.md` + `base/ai-tells.md`
> **Adds**: 정보성 문서 특화 규칙 (따라하기 용이성, 섹션 독립성)

정보성 문서는 독자가 절차를 따라하거나 개념을 참조하기 위한 매체다.
독자가 처음부터 읽지 않고 **원하는 섹션부터 점프**할 수 있어야 한다.

---

## 적용 대상

| 매체 | 적용 |
|------|------|
| 설정 가이드·트러블슈팅 (블로그·위키) | 본 익스텐션 |
| API 사용 가이드 | 본 익스텐션 |
| 절차 안내 (배포·릴리스·온보딩 등) | 본 익스텐션 |

---

## I1. 섹션 독립성

독자가 헤딩으로 점프했을 때, 그 섹션만 읽어도 작업이 가능해야 한다.

- 각 H2 섹션의 첫 문장이 해당 섹션의 목적을 정의 (`readability.md P3` 강화)
- 다른 섹션의 변수·결과를 가정하지 않음 (필요하면 명시적으로 링크)
- 코드 블록은 그 섹션 안에서 완결 (이전 섹션의 변수를 참조하지 않음)

---

## I2. 절차형 글의 순서 목록 우선

따라하기 절차는 순서 목록(`1.`)을 사용한다.
비순서 목록(`-`)은 항목 간 순서가 없을 때만 쓴다.

Bad:
```markdown
- 설정 파일을 연다
- 값을 변경한다
- 저장한다
- 서버를 재시작한다
```

Good:
```markdown
1. 설정 파일을 연다
2. 값을 변경한다
3. 저장한다
4. 서버를 재시작한다
```

---

## I3. 트러블슈팅: 증상 → 원인 → 해결 표

트러블슈팅 문서는 증상 기반 인덱스를 표로 제공한다.

```markdown
| 증상 | 원인 | 해결 |
|------|------|------|
| `Connection refused` | DB 컨테이너 미기동 | `docker compose up -d db` |
| `Permission denied` | 볼륨 마운트 권한 | `sudo chown ...` |
```

테이블 열 수는 `readability.md L4` 에 따라 5개 이하로 유지.

---

## I4. 정리 섹션 = 핵심 요약·참고 자료

본문 끝에 정리 섹션을 둔다. TL;DR 과 다르게 본문을 다 읽은 독자를 대상으로 한다.

```markdown
## 정리

- 핵심 명령어: `...`
- 추가 참고: [공식 문서](...)
```

---

## 필수 구조

```markdown
## TL;DR (선택 — 1500자 이상 시 필수)
## 개요 (무엇을, 왜)
## 본문 (절차 또는 개념 설명, H2 섹션 단위로 독립적)
## 정리 (I4 — 핵심 요약·참고 자료)
```
