# 이슈 익스텐션 (골격)

> **Base**: `base/readability.md` + `base/tone.md` + `base/punctuation.md` + `base/ai-tells.md`
> **Adds**: 사내 이슈(Dooray·Jira·GitHub) 작성 특화 규칙

> ⚠️ 골격만 작성. 사내 이슈 작성 빈도가 쌓이면 규칙을 채운다.

연계 스킬:
- `toolkit:dooray-api` (Dooray 태스크 조회·수정)

---

## 적용 대상

| 매체 | 적용 |
|------|------|
| Dooray 태스크 본문 | 본 익스텐션 |
| GitHub Issue (사내·퍼블릭) | 본 익스텐션 + 퍼블릭 시 `blog.md B6` 메타 일부 차용 |

## (예정) ISS1. 재현 절차 명시

증상·재현 환경·재현 단계·기대 동작·실제 동작 5요소를 표 또는 섹션으로 명시.

## (예정) ISS2. 영향 범위 명시

해당 이슈가 영향을 미치는 사용자·시스템·데이터 범위를 명시.

## (예정) ISS3. 우선순위·라벨

조직 컨벤션에 따라 우선순위·라벨을 부여.
