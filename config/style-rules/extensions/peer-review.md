## PR1. 사실 기반 강점·개선점

추상적 평가를 금지한다. 구체 사례·관측 가능한 행동·결과를 기반으로 한다.

Bad (추상 평가):
```
협업이 좋다. 책임감이 강하다.
```

Good (사실 기반):
```
강점: PR 리뷰 응답이 평균 2시간 이내. 4건의 장애 대응을 자발적으로 주도.
```

`ai-tells D-4` (`혁신적인`, `차세대`) 같은 과장 형용사를 base 보다 강하게 차단.

---

## PR2. 균형 — 강점·개선점 1:1 또는 1:0.5

강점만 또는 개선점만 나열하지 않는다. 일반적으로 강점 N개, 개선점 N개 또는 N/2개.

극단적 비율(강점 5개·개선점 0개)은 평가의 형평성을 떨어뜨린다.

---

## PR3. 개선점은 행동 변화 가능 표현

개선점은 "행동 변화가 가능한 표현"으로 작성한다. 인격 평가 회피.

Bad (인격 평가):
```
A 는 소극적입니다.
```

Good (행동 가능):
```
회의에서 의견 표명 빈도가 낮습니다. 사전에 의견을 글로 정리해 공유하면 토론 진입이 쉬워질 것 같습니다.
```

---

## PR4. 평가 어휘 회피

조직 평가서 특유의 과장 어휘를 회피한다.

| 회피 | 대체 |
|------|------|
| `탁월한`, `뛰어난` | 구체 사례 직접 기술 |
| `리더십이 있다` | 어떤 상황에서 누구를 어떻게 이끌었는지 |
| `긍정적인 마인드` | 구체 행동 (예: "어려운 상황에서도 회고를 주도") |
| `소통이 좋다` | 매체·빈도 (예: "Slack 응답 평균 30분 이내") |

`tone.md T11` (권위 용어 남발 금지) 의 강화 버전.

## PR5. 시제·주어

- 시제: 평가 기간의 사실은 과거형 (`~했습니다`, `~를 주도했습니다`)
- 주어: "A 는" / "팀원 A 는" / "함께 일한 B 는" 명시. 익명 평가는 "함께 일한 동료는"
